## TESTS
Here you can find all the scripts used in the early stages of the project to test the scaling action and the forecasting mechanism. These scripts were used to validate the functionality of the DREEM controller and to ensure that the scaling actions were performed correctly based on the forecasts generated by the `forecast` script.
A YOLO docker image is used together with Locust to create time-varying workload on the node.

Final tests and validation have been performed on a bare metal cluster using Metal3 as infrastructure provider and MuBench as benchmarking tool to generate workload on the cluster and to validate the scaling actions performed by the DREEM controller.
